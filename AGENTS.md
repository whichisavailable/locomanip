# RobotLab Conventions

## Scope
- 只允许修改robot_lab目录下的文件，在用户明确开始写之前先跟用户讨论清楚，不要上来就开始改代码。

## Links
- 回答里引用文件时，不要使用网页式链接。优先提供可在 IDE 中直接定位的文件路径，并尽量附行号。
- 在当前终端渲染器里，如需提供可点击的本地文件链接，使用 markdown 本地绝对路径格式：`[label](/c:/abs/path/to/file.py:12)`，不要把 `C:/...` 直接写进链接目标，否则容易被当成网页 URL。


## Comments
- 新增代码要写详细的中文注释，最好每一行都有注释，重点解释量的语义、设计意图和为什么这样做。
- 不要为了“看起来更完整”擅自加入用户未明确要求的门控、启发式条件、额外逻辑。
- 修改已有实现时，优先把新加部分注释清楚，不要求顺手重写整文件历史注释。

## Encoding
- 默认按 UTF-8 处理文件。
- 修改时避免引入乱码、编码破坏或风格明显不一致的注释。
- 如果发现历史乱码，除非用户明确要求，否则不要顺手做大范围编码清理。
